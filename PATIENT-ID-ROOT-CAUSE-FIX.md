# 🔧 PATIENT ID ROOT CAUSE FIX

## Problem Statement
Patient IDs are inconsistently used across the application:
- Some places use UUIDs (`88aaa7f4-e119-4985-96ca-cbdb9922bd5d`)
- Some places use legacy IDs (`MEC0017`)
- URLs sometimes contain UUIDs instead of legacy IDs
- Reports sometimes show UUIDs
- Database queries are inconsistent

## Root Cause
1. **No centralized normalization**: Each function handles patient IDs differently
2. **Inconsistent data flow**: Patient objects sometimes have UUID as `id`, sometimes legacy ID
3. **Missing normalization in URLs**: Navigation functions don't always normalize before creating URLs
4. **Database schema confusion**: `id` (UUID) vs `patient_id` (legacy ID) not consistently used

## Solution: Centralized Normalization System

### 1. Created `js/patient-id-normalizer.js`
Centralized functions for patient ID normalization:
- `normalizePatientId()` - Normalizes any identifier to legacy ID
- `normalizePatientIdForUrl()` - For URL generation
- `normalizePatientIdForDisplay()` - For display/reports
- `normalizePatientIdForQuery()` - For database queries (returns UUID)

### 2. Database Schema Rules
- **`id` column**: UUID (auto-generated by Supabase) - INTERNAL USE ONLY
- **`patient_id` column**: Legacy ID (MECXXXX format) - USER-FACING
- **Rule**: ALWAYS use `patient_id` for:
  - URLs
  - Display
  - Reports
  - User-facing operations
- **Rule**: ONLY use `id` (UUID) for:
  - Database queries (foreign keys)
  - Internal operations

### 3. Patient Object Structure
```javascript
{
  id: "MEC0017",              // Legacy ID (for backward compatibility)
  patient_id: "MEC0017",      // Legacy ID (primary)
  _supabaseUuid: "88aaa7f4-...", // UUID (for database queries)
  // ... other fields
}
```

### 4. Fixes Applied

#### A. Navigation Functions
- `viewPatient()` in `js/patients-supabase.js` - Uses normalizer
- All `window.location.href` with patient IDs - Normalized
- All `onclick` handlers with patient IDs - Normalized

#### B. URL Generation
- Patient detail links: `patient-details?id=MEC0017` (not UUID)
- Clinical note links: `clinical-note?patientId=MEC0017`
- Appointment links: `add-appointment?patientId=MEC0017`
- Encounter links: `patient-encounters?patientId=MEC0017`

#### C. Display Functions
- All patient ID displays use `getPatientIdentifier()` or `normalizePatientIdForDisplay()`
- Reports use legacy IDs
- Tables use legacy IDs

#### D. Database Queries
- Queries use UUID (`id` column) internally
- Results are converted to legacy ID before display
- `resolvePatientByIdentifier()` handles UUID → legacy ID conversion
